# Phase 7 — Check-ins and Graph Revisions

## Goal

Let the user record actual progress and revise the Life Map.

## Non-goals

- No automated behavioral tracking.
- No deterministic future claims.

## Deliverables

- Check-in form.
- Check-in list.
- Revision proposal generator.
- Graph diff model.
- Accept/reject revision flow.
- Route status tracking.

## Acceptance criteria

- User can create a check-in.
- System proposes map changes based on check-in.
- User can inspect diff before accepting.
- Accepted revision creates a new snapshot.
